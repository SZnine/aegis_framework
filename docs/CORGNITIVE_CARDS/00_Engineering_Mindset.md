最小单元模块的建立方式是：`设计一个NAME1.0，作为模块文件`

对最小单元模块扩充的方式是：`NAME深入——增加A功能`

建立核心文件的方式是：`建立一个A功能的NAME,完成A_eg功能并完成测试`

最核心文件的扩充的方式是：`添加（完善）A_eg功能（方法）`



**思考**

一个需求如何转变为一个项目：
需求 -> 最小需求 -> 代码实现  -> 最小单元 -> 扩充 -> 成品
在上述中，利用的便是分解一个需求为核心+模块，二者都以最小化单位为基础材料，通过深化扩充的方法，构建成品结构，最后完成拼接，完成后进行测试，其实在最小单元时，就应该进行一次测试，用以对逻辑完整性和代码可行性的验证。
